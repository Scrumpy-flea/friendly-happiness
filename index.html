Html For Beginners!

Html stands for HyperText Markup Language. Html is primarly used for adding information to a webpage. I believe it is the first step towards creating a website. An Html element is a piece of content in a html document that uses the following syntax:opening tag, content, closing tag.

When starting a HTMl document for the first time, you need to start with <!DOCTYPE html>. This tells the brower that you are working in the current version of HTML.

Then you need to place all of your code, inside another HTML tag. <html> </html> The brower understands that all the code within those tags is HTML.

<p>Is the opening tag.

Hello World! Is the content

</p>Is the closing tag

Html is organsied into a family tree type of structure, also called nesting. Elements that sit within other elements are the child of the bigger element. For example, the h1 element is the child of the element it is fitted into. Usually, the body or div elements.

  • Some html tags have a specific purpose. eg, image or video tags.
  • Most tags just describe the content, this helps us with styling later on.
  • There are an infinate number of tags avaible
  • Knowing when to use each one is based upon how you want to describe your html content.
  • Descriptive, well chosen tags are the key to high quality web development.

Element
In html an element is the opening, closing tag and the content that lies between them. The term tag is used interchangably with the word element.

Attribute
Attributes tell us more about the content of the element. They are values added to the opening tag of an element. They change the elements behaviour. They appear in the opening tag of the element and are made up of 2 parts. a name and a value. These are seperated by =. The attributes name indicates what kind of extra information you are supplying. The value is the setting for the name. It should be placed in "" Most attributes can only be used on certain elements, but some are interchangable.
Here is an example:
<p lang="fr">Paragraphe en Francais</p>
The attribute is lang, this tells us the language the element (paragraph) is in. The value of the attribute is fr, specifing it is in french.

Css
Css stands for cascading style sheet. Css is a different language to html. You can do limited css in html, but it is better to keep the code in seperate documents. Css is used to style a html document.

Syntax
A Set of rules that define the combinations of symbols to be correctly structured. Each coding language has its own syntax.

Semantic HTML
Semantic elements provide information about the content between the opening and closing links. For exmaple <header>, <nav>, <artical>, <figure> and <footer> are all semantic elements. Making the code easier to read for anyone else reading it.

Non Semantic html
Non Semantic html is when an element does not describe what is contained inside it. For example <div> can be used to contian paragraphs or our header information. Making the code harder to read and edit. <span> is also a non semantic element.

Validation
Checking user provided data against required data. Validation is used when submitting a <form> For example, username and password. There are different types of validation.

Client Side Validation
If we want to check the data sent from a form. This usually occurs before the data is sent to the server. Different browser do this differently, but the outcome is always the same. HTML5 do build in client side validation. This is good because it protects the server from bad code, or code which could be submitted by a bad person. It also saves time as we don't have to wait for the data to sent there and back. Also, quick feedback is availble for forms. So if a user submits a form with the information filling the required fields, they can quickly get it back without having to redo the form.

Server side validation
Data is sent to another server for processing. For example, a login page.
href
Hyperlink reference. href is used to link to the address where the file is located.
Relative Path
Relative path is the file name that shows the path to a local file. (a file on the same website.)
Absolute path
A full URL. Which is stored in a different folder.
DOM
Document Object Model. This is the hierarchy all HTML follows. For example: parent element <body>l; then has a child element, <H1>. The DOM holds all our elements, in the order that we have placed them.

  • DOM = Document Object Model
  • Code in html is organised into a family tree.
  • When an element in nested into another element it becomes the child of the element it is nested into.

For example:

Linking.

Linking to other webpages.

  • Add links to other webpages using an anchor tag. <a> </a>
  • Include the text you want the link rendered in between the anchor tags.
  • Needs a href attribute with the link. As is this linking to an absoulte URL.
  • Want a new window, so your webpage isn't lost when the user clicks your link? Use the target attribute and give it a value of _blank.
  • You could put image as the clickable link. Just insert it between the anchor tags.

<a href="https://www.wikipedia.org/" target="_blank"> This is a link to wikipedia.</a>

Linking to a relative page.

  • Linking to a relative page is just linking internal pages. (home, contents etc)
  • for multi-page websites, files are stored in a "root directory" or a main directory.
  • Html files are to be stored in the same folder. We can link these together with a relative path.
  • We need to tell the browser the relative path. Basically, which folder's it needs to look in to find the relative page.
  • Anchor tag is used with a href attribute.
  • Using a ./ in the link tells the browser to look in the current folder.
  • Don't forget to add a target attriubute of _blank. This will open the link in a new tab.

<a href="./contact.html" target="_blank">Contact</a>

Adding Media

Images

  • <img> is a self-closing tag.
  • src is a required attribute. src needs to be set to the location of the image.
  • The / at the end of the code doesn't have to be there for it to render correctly. It's just good etiquette.
  • alt is another attribute that can be used. alt means alternative text. Bringing meaning to the image if it won't display properly or a ereader visits the website.
  • alt should be a discription of the image.
  • Using alt helps search engines find your image. Improving the ranking of the website.

<img src="Resources/dustin-humes-IyP6Y7LpOWY-unsplash.jpg" alt="Photo by Dustin Humes on Unsplash."/>

Videos

  • Video element requires an opening and closing tag.
  • <video> tag requires a src attribute, with a link to the source.
  • The source can be a file hosted with the site, or a URL that links to a video hosted on another site.
  • Width and Height attributes are used to set the size displayed.
  • Some attributes that can instruct the browser to alter playback:
    • controls: Play, pause, volume control and full screen option.
    • auto-play: Playing as soon as the webpage is loaded.
    • loop: Continuously on repeat.

<video src="coding.mp4" controls>Video not supported</video>


Audio

  • <audio> element is used to embed audio into a document.
  • Use the source attribute to link to the video.
  • Using the type attribute to specify what kind of video it is will help the browser with loading the audio. As the browser will know if it can support it.
  • Some other attributes for <audio> are:
    • controls: Shows the controls play and mute.
    • src: specifies the URL of the audio file.
    • autoplay: immediatly plays the audio once the webpage has loaded.

<audio>
   <source src="anaudiofile.mp3" type="audio/mp3">
</audio>


Embed

  • The embed element can be used for any media content. eg, video, audio, gifs from an external source.
  • This means that websites that have an embed button have some form of media content that can be added to other websites.
  • <embed> is a self-closing tag, unlike <video>.
  • embed can be used to add local files as well as media content from some other websites.
