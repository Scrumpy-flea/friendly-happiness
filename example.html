Forms

  • A form can be any type of input.
  • Forms are sent over the internet, which is essentially a network of computers that send and recieve informertion.
  • The hhtp request instructs recieving computer how to handle incoming information.
  • The form element is great for collecting information.
  • The element needs to be supplied with the location of where the form information will go, and what http request to make. eg:

<form action="example.html" method="POST">
   </form>

  • Above is the skeleton of a form. It will send information to example.html as a post request. POST doesn't have to be in capitals, but it is good etiquette it do so.
    • action decides where the information submitted is sent.
    • method is assigned a http verb that is included in the http request. It determines how information is sent and passed.
  • <form> can contain child elements. Eg, a header (What the form is about, or even a paragraph for more detail). For Example:

<form action="/example.html" method="POST">
   <h1>Creating a form.</h1>
   <p>More information to read.</p>
</form>

  • The code above included no user input, but it is the foundations of a form.

Input

  • <input> now needs a type. This determines how it renders on a page and what type of data it can accept.
  • name is also needed. Otherwise the input won't be sent when the form is submitted.
  • So what is typed into the box is sent with the name value. Typed as text when the form is submitted.
  • So if 'important details' where typed into the box and sent. The text woule be:
    'first-text'='important details' is sent to /example.html
  • Because, the value of name is 'first-text' and the value of the value (textbox) is 'important details'.
  • You can assign a default value for the value attribute. So a pre-filled text field appears.

<form action="/example.html" method="POST">
   <input type="text" name="first-text" value="already pre-filled">
   <h1>Creating a form.</h1>
</form>


Labels

  • We also need a label, so users know what they need to input.
  • Label has opening and closing tag. What is inbetween those tags renders on the webpage.
  • To associate the label and the input. We need to give the input an id.
  • We then assign the label a for. This tells the browser to link them together.

<form action="/exampla.html" method="POST">
   <label for="meal">What do you want to eat?</label>
    <br>
   <input type="text" name="food" id="meal">
</form>

Password

  • When input type is set to password, the user doesn't see the characters they are inputing, they only see *.
  • When the information is being sent, it is sent as letters not the * the user can see.

<form>
   <label for="pin">Please enter your pin:</label>
   <input id="pin" name="pin" type="password" >
</form>

Number input

  • Allows only numbers and a few special characters. (Like -, + and .).
  • Use a step attribute, which creates arrows inside the input field to increase or decrease the value by our chosen step value.

<form>
   <label for="years">Years of experience</label>
   <input id="years" name="years" type="number" step="1">
</form>

Which renders...

  • Or by changing the type to range we can render a slider.
  • We need to add min and max values. This determines how low / high the number inputted can go.
  • Also the step attribute determines how big or little jumps the slider makes.
  • Eg,

<form>
   <label for="years">Years of experience</label>
   <input id="years" name="years" type="range" min="0" max="20" step="1">
</form>

When the slider moves by one, the value of the input's value attribute changes.

Checkbox

  • Set the type value in the input element to checkbox.
  • Need to write an invidiual input and label tags for each checkbox.
  • Make sure you apply the same value for every "name" attriubute.
  • Also include a value attriubute for each checkbox. This is what information is sent with the form. (This won't be rendered on the page, Must use that label.)
  • Here is an example:

<form>
   <p>Pizza Toppings</p>
    <label for="cheese">Cheese</label>
    <input id="cheese" name="topping" type="checkbox" value="cheese">
    <br>
    <label for="tomato">Tomato</label>
    <input id="tomato" name="topping" type="checkbox" value="tomato">
    <br>
    <label for="mushroom">Mushrooms</label>
    <input id="mushroom" name="topping" type="checkbox" value="mushroom">
    <label for="pepper">Peppers</label>
    <input id="pepper" name="pepper" type="checkbox" value="pepper"> </form>

  • Notice we didn't add the break, between mushrooms and peppers. The browser renders the final checkbox next to the previous one, rather than underneith.

Submit Button

  • In a form, use input and set type to submit.
  • Whatever value you assign appears on the button.
  • Here is an example:

<form>
   <input type="submit" value="Send">
</form>
